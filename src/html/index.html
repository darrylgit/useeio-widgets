USEEIO Widgets

Basic principles

Each widget is a component that is created with a configuration object with a reference to a USEEIO model, a selector of an HTML element (e.g an ID or class of a div element) where the widget should be mounted, and other widget specific properties. The USEEIO model is created from an API endpoint. If the endpoint just provides cached JSON files, the field asJsonFiles should be set to true when creating the model: 

            
    var model = useeio.model({
        endpoint: './api',
        model: 'GAUSEEIO',
        asJsonFiles: true,
    });

Such a model caches resources like sectors and matrices that it fetches from the API so that the widgets should use the same model instance.

The communication between the widgets is also done via configuration objects and a set of methods that all widgets implement. Calling the update method will (re-)render the widget with the given configuration: 

            
    widget.update(config);

There are widgets that create new configurations (like the SectorList widget). With the onChanged method it is possible to react on these configuration changes: 

            
    widget.onChanged((config) => {
        // do something with the updated config
    });
